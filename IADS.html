Avionic Display System

Display Interfacing & Intercommunication between Two Microcontrollers through RS-422 protocol using Bare Metal Programming

Unveiling the Inner Workings: A Closer Look at the Project's Operations

  • To establish communication with the peripherals, namely the Vacuum Fluorescent Display (VFD) and Joystick, the necessary GPIO (General-Purpose Input/Output) pins are configured. GPIO pins provide the interface between the microcontroller and external devices. In this case, they enable communication and control between the slave computer and the VFD and Joystick.
  • For debugging purposes, USART communication is utilized to transmit data from the slave computer to a terminal program such as Putty. This enables real-time monitoring and display of the transmitted data, facilitating debugging and analysis of the system.
  • The slave computer is configured to receive and analyze requests from the keyboard and joystick. This involves processing the input signals from the peripherals and interpreting them to determine the corresponding actions or commands to be executed.
  • A Control Block, consisting of a 15-byte packet of data, is sent from the master computer to the slave computer continuously. The Control Block contains instructions and commands to be executed by the slave computer.
  • Conversely, the slave computer sends an 8-byte status block to the master computer. This status block includes important information such as the Echo word (confirmation of received data), keyboard functions, PDU (Power Distribution Unit) status, Discrete word (digital input/output status), Reserved bit, Failure indication, and Checksum for data integrity.
  • On the master computer, the function name and other relevant data are extracted from the received 8-byte status block. This allows the master computer to understand the current status and conditions reported by the slave computer.
  • Based on the received status block, the master computer sends the appropriate commands and instructions within the Control Block to the slave computer. These commands may involve actions to be performed, settings to be changed, or data to be transmitted to external systems.
  • The instructions sent to the slave computer are then displayed on the VFD. The VFD serves as a user interface, providing visual feedback and instructions to the system operator or user
  • Finally, the data accessed by the slave computer is transmitted to an externally connected FPGA (Field-Programmable Gate Array) system. The FPGA system can further process and manipulate the data to perform complex tasks or interface with other external devices.
